# DESIGN.md

## Цель интерфейса

Расширение должно выглядеть как естественная часть Mattermost: кнопка стикеров находится внутри панели действий основного композера и тредов между emoji picker и загрузкой файлов, а выбор стикера занимает минимум внимания.

## Размещение

- Кнопка монтируется в action row каждого найденного композера, включая панель тредов, между `emojiPickerButton` и upload-кнопкой.
- Панель выбора не является дочерним элементом action row. Она добавляется в `document.body` и позиционируется рядом с кнопкой через `position: fixed`.
- Такой подход защищает панель от `overflow: hidden`, `transform` и других ограничений контейнеров Mattermost.

## Иконка

Иконка кнопки — контурный стикер с загнутым уголком и улыбкой. Она сделана inline SVG, чтобы не добавлять отдельные ассеты и сохранять цвет через `currentColor`.

Состояния:

- Default: нейтральный цвет Mattermost.
- Hover/focus: светлая синяя подложка и акцентный синий контур.

## Панель стикеров

Панель состоит из двух зон:

- Левая колонка: первая вкладка `Популярные`, затем папки стикерпаков.
- Правая зона: сетка стикеров выбранной папки.
- Нижняя строка: маленькие чекбоксы `Моментальная отправка` и `bigstic`, а также короткий статус операции.

Текущий пак подсвечивается только в списке папок; название пака не дублируется в шапке. Список паков текстовый и компактный, без декоративных иконок. Сетка использует фиксированную высоту строк и `object-fit: contain`, чтобы большой список стикеров скроллился, а не сжимался по вертикали. Пустая вкладка `Популярные` показывает короткое состояние-заглушку.

## Поведение

- Клик по кнопке открывает или закрывает панель.
- Для мыши и трекпада панель открывается на `pointerdown`, чтобы обработчики Mattermost вокруг toolbar не успевали перехватить обычный `click`.
- `Command + Option + S` открывает или закрывает панель стикеров. Повторные события при удержании клавиш игнорируются.
- В DOM должен существовать только один `#band-stickers-panel`; кнопок может быть несколько, по одной на композер.
- Клик вне кнопки и панели закрывает панель.
- `Escape` закрывает панель.
- При `scroll` и `resize` позиция панели пересчитывается.
- После выбора стикера панель закрывается, а файл передается в активный композер Mattermost как вложение.
- По умолчанию отправляется PNG 128x128 из `smallPath`.
- Если `bigstic` включен, отправляется оригинальный ассет из `path`.
- WebP-ассеты перед отправкой конвертируются в PNG, чтобы снизить риск отказа загрузки на стороне Mattermost.
- На время прикрепления файла активный композер получает невидимый символ `U+200B`, затем символ сразу удаляется. Это имитирует ввод текста и помогает Mattermost принять вложение.
- Если `Моментальная отправка` включена, расширение ждёт активную кнопку отправки до 5 секунд и кликает её. Если кнопка не стала доступна, стикер остаётся в композере для ручной отправки.
- После успешного прикрепления стикера счётчик использования сохраняется в `localStorage`; вкладка `Популярные` сортируется по частоте и времени последнего использования.

## Ограничения

- Расширение работает только на `https://band.wb.ru/*`.
- Список стикеров задается явно в `src/stickers.js`, потому что Chrome extension не читает директории ассетов динамически.
- Новые папки стикеров должны добавляться в `window.BAND_STICKER_PACKS` и `web_accessible_resources` в `manifest.json`.
- Пути extension-ресурсов должны использовать ASCII-имена папок, чтобы избежать расхождений Unicode-нормализации в Chrome.
- Для каждого стикера нужен оригинальный `path` и маленький `smallPath` 128x128.
- Поддерживаемые исходники: WebP, PNG, JPG/JPEG. `.tgs` не подключается без отдельного рендера Lottie.

## Автопроверки

- Консистентность: каждый стикер из `src/stickers.js` должен ссылаться на существующие файлы, `smallPath` должен быть PNG 128x128, а пак должен быть отражен в README и HTML-документации.
- Безопасность: manifest остается MV3, домен ограничен `https://band.wb.ru/*`, широкие permissions не добавляются, в исходниках запрещены `eval`, `new Function`, `document.write` и прямые сетевые `fetch` на внешние URL.
